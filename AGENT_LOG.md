# AGENT_LOG.md · 关键决策记录

## 项目概述
- 项目名称：小小动物园 · 亲子任务打卡系统
- 架构：Python Flask + SQLite + Jinja2（服务端渲染）
- 主题：动物园员工，孩子通过完成任务获得积分和成长

## 关键决策

### 1. 技术栈选择（2026-06-28）
- **选择**：Python Flask + SQLite + Jinja2
- **理由**：项目要求 Web 应用且不允许前端框架，Flask 轻量简洁，适合单人开发。SQLite 无需安装数据库服务，部署简单。
- **替代方案**：Django（太重）、Node.js（用户不熟悉）

### 2. 盲盒积分机制（2026-06-28）
- **选择**：基准分 ± 随机波动，孩子只看最终分数，不知基准分
- **理由**：保持"盲盒"惊喜感，同时让更难的任务有更高基准分（通过波动范围调节）
- **实现**：`calculate_blind_score()` 函数，最低分保证为 1

### 3. 双货币系统（2026-06-29）
- **选择**：积分（用于兑换活动） + 金币（用于宠物系统）
- **理由**：积分是主线成就感来源，金币是宠物系统独立货币，避免两个系统互相抢夺同一资源

### 4. 双账号分离（2026-06-29）
- **选择**：家长和孩子使用各自独立账号登录
- **理由**：每次使用都要选人，各账号拥有独立的设置和数据视角

### 5. 宠物成长阈值（2026-06-29）
- **选择**：蛋阶段喂食 8 次 → 幼年阶段 23 次喂食 + 10 次玩耍 → 成年
- **理由**：成长周期适中，孩子能在几天内体验到完整流程

### 6. 数学游戏 4 年级分级（2026-06-29）
- **选择**：幼儿园（10以内）→ 一年级（100以内无进退位）→ 二年级（九九乘除）→ 三年级（1000以内/两位数乘整十/带余除法）
- **理由**：覆盖 4-8 岁年龄段，每个台阶有明确的知识点进阶

### 7. 勋章系统惰性检查（2026-07-07）
- **选择**：孩子访问勋章页面时触发检查，而非打卡时实时触发
- **理由**：避免每次打卡都做全量检查，降低性能开销

### 8. 心愿单进度计算（2026-07-07）
- **选择**：动态从可用积分计算，而非手工标记
- **理由**：自动同步，无需孩子手动管理进度；每次访问心愿单页面自动更新

### 9. 容器分发（2026-07-07）
- **选择**：Docker 容器作为分发方式
- **理由**：作业要求容器分发，Docker 是最普遍的标准

## 已知限制
- 7 天连续打卡检查使用启发式方法（检查最近 7 天日期是否存在），非严格连续
- 系统无真实时间调度（宠物成长依赖操作次数而非真实时间）
- 无数据库迁移机制（开发阶段直接重建）
