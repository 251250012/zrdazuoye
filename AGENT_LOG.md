# AGENT_LOG.md · 关键决策记录

## 项目概述
- 项目名称：小小动物园 · 亲子任务打卡系统
- 架构：Python Flask + SQLite + Jinja2（服务端渲染）
- 主题：动物园员工，孩子通过完成任务获得金币，养宠物、做答题

## 关键决策

### 1. 技术栈选择（2026-06-28）
- **选择**：Python Flask + SQLite + Jinja2
- **理由**：项目要求 Web 应用且不允许前端框架，Flask 轻量简洁，适合单人开发。SQLite 无需安装数据库服务，部署简单。
- **替代方案**：Django（太重）、Node.js（用户不熟悉）

### 2. 盲盒积分机制（2026-06-28）
- **选择**：基准分 ± 随机波动（20%），孩子只看最终金币
- **理由**：保持"盲盒"惊喜感，同时让更难的任务有更高基准分
- **变更**：后来改为仅保留金币，去掉积分制；波动自动 ±20%，家长只需设基准分

### 3. 从双货币简化为单货币（2026-07-08）
- **选择**：去掉积分制，只保留金币
- **理由**：家长懒得配置兑换活动，商城系统闲置；金币统一用于宠物养成
- **影响**：删除了活动管理、商城、兑换、美好时刻等全部页面和代码

### 4. 双账号分离（2026-06-29）
- **选择**：家长和孩子使用各自独立账号登录
- **理由**：每次使用都要选人，各账号拥有独立的设置和数据视角

### 5. 宠物成长阈值（2026-06-29）
- **选择**：蛋阶段喂食 8 次 → 幼年阶段 23 次喂食 + 10 次玩耍 → 成年
- **变更**：去掉了宠物数学游戏，蛋阶段只显示喂食按钮

### 6. 宠物系统改进（2026-07-08）
- **选择**：支持同时养多只宠物（最多5只），宠物蛋200金币购买
- **理由**：用户希望同时养育多个宠物，增加可玩性

### 7. 宠物图片系统（2026-07-08）
- **选择**：每种宠物幼年3张 + 成年3张真实图片，用宠物ID固定选图
- **理由**：用户提供了宠物图片，替代 emoji 展示，每只宠物长相不同

### 8. 数学游戏独立为答题系统（2026-07-08）
- **选择**：从宠物互动中独立出来，每天10轮，每轮最多10题，答错结束
- **理由**：用户希望数学题独立于宠物，每天有限次，答对×10金币

### 9. 勋章系统惰性检查（2026-07-07）
- **选择**：孩子访问勋章页面时触发检查，而非打卡时实时触发
- **理由**：避免每次打卡都做全量检查，降低性能开销

### 10. 用户名与密码规则（2026-07-09）
- **选择**：家长可重名，密码全系统唯一；同一家长下孩子不能重名
- **理由**：多个家长可能使用相同用户名，但密码必须不同以防止登错号

### 11. 数据库并发防护（2026-07-09）
- **选择**：启用 WAL 模式 + busy_timeout + 关闭 debug reloader
- **理由**：解决 Railway 部署时 "database is locked" 错误

### 12. PWA 支持（2026-07-08）
- **选择**：添加 manifest.json、service worker、SVG 图标
- **理由**：用户希望网站能作为 App 添加到手机桌面

### 13. 容器分发（2026-07-07）
- **选择**：Docker 容器作为分发方式
- **理由**：作业要求容器分发，Docker 是最普遍的标准

## 已知限制
- 7 天连续打卡检查使用启发式方法（检查最近 7 天日期是否存在），非严格连续
- 系统无真实时间调度（宠物成长依赖操作次数而非真实时间）
- 无数据库迁移机制（开发阶段直接重建）
- 免费云部署无持久化存储，每次重新部署数据库重置
