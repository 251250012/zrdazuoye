# REFLECTION.md · 项目反思

## 1. 项目整体评价

小小动物园亲子任务打卡系统从构思到实现，覆盖了 5 大功能模块：任务打卡、金币盲盒、宠物养成、答题系统、勋章墙。项目达到了预期目标——一个真实可运行、功能完整的 Web 应用，已部署到云端可通过公网访问。

## 2. 做得好的地方

### 2.1 设计阶段的用户参与
用户全程参与设计决策，从功能选择到界面元素（如"动物园员工"主题、神秘卡片上的"？"）都由用户确认。避免了"开发出来不是用户想要的"的问题。

### 2.2 Subagent-Driven Development 效率高
将任务分拆给独立子代理实现，每个任务完成后立即审查，有效避免了问题堆积。代码审查机制在大作业中起到了实效。

### 2.3 盲盒金币机制的设计
基准分 + 自动 ±20% 随机波动的设计既保持了游戏性，又让家长可以通过任务难度调节奖励。孩子不知道基准分的设定增加了神秘感。

### 2.4 宠物系统的完整性
5 种宠物各有 3 种互动 + 3 个成长阶段 + 15 种反应文案，加上真实图片展示，内容充实。孩子能体验到从蛋到成年的完整养育流程，且可同时养多只宠物。

### 2.5 持续响应用户反馈
开发过程中用户多次提出功能变更（去商城、加答题、改宠物规则），项目灵活调整，最终符合用户真实需求。

## 3. 可以改进的地方

### 3.1 缺乏自动化测试
项目仅有少量测试，更多功能缺乏自动化测试覆盖。如果是一个长期维护的项目，应该为重点功能（尤其是金币计算、勋章检查）编写单元测试。

### 3.2 无数据库迁移机制
当前 SQLite schema 变更需要删除数据库重建。对于生产环境，应该使用迁移工具（如 Alembic）。

### 3.3 7 天连续打卡检查是启发式
当前使用日期集合成员检查，非严格意义上的连续。在严格场景下需要更精确的算法。

### 3.4 响应式设计不够完善
UI 在移动设备上可能需要额外适配。当前主要面向桌面浏览器。

### 3.5 安全方面
- 缺少会话过期机制
- 缺少 CSRF 保护
- 没有登录失败次数限制

## 4. 作为大一新手的收获

- 初步掌握了 Flask Web 开发、SQLite 数据库操作、HTML/CSS 前端开发
- 体验了 AI 辅助开发的工作方式，了解了 AI 的优势和局限
- 认识到真实项目中需求会不断变化，代码需要灵活应对
- 学会了 Docker 部署和云服务发布的基本流程
