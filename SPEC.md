# SPEC.md — 亲子任务打卡 & 奖励系统

> **项目名称**：小小动物园 · 亲子任务打卡系统  
> **版本**：v2.0  
> **日期**：2026-07-09

---

## 1. 问题陈述

### 1.1 要解决什么问题

家中有小学/幼儿园阶段的孩子，家长希望：
- 培养孩子良好的日常习惯（读书、写字、练琴等）
- 让孩子理解"努力就有回报"的概念
- 减少家长与孩子之间关于"做没做"、"能不能玩"的拉扯
- 让亲子互动更有趣、更有仪式感

### 1.2 目标用户

- **主要用户**：6-10 岁儿童（幼儿园中大班 ~ 三年级）
- **管理用户**：孩子的父母/监护人
- **每个家庭独立使用**，家长账号隔离，孩子归属于指定家长

### 1.3 为什么值得做

将"习惯养成"与"游戏化"结合，以"动物园"主题包装金币系统，配合盲盒机制、宠物养成、答题闯关等元素，让孩子在趣味中培养自律。家长端提供完整的任务配置与管理能力，同时支持 PWA 可添加到手机桌面使用。

## 2. 用户故事

> 遵循 INVEST 原则（Independent, Negotiable, Valuable, Estimable, Small, Testable）

1. **作为孩子**，我希望每天看到今天有哪些任务可以做，打卡后能立即看到我获得了多少金币，这样我有动力去完成任务。

2. **作为孩子**，我希望养一只属于自己的宠物，给它喂食、玩耍、摸它，每次互动都有不同的反应，这样即使一个人也能获得陪伴感。

3. **作为孩子**，我希望看到我获得的勋章墙，这样我能感受到自己在不断进步。

4. **作为孩子**，我希望每天可以做数学题赚金币，答对越多金币越多，这样学习变得有趣。

5. **作为孩子**，我希望同时养多只宠物，每只都有自己的长相，这样我可以收集不同动物。

6. **作为家长**，我希望可以管理任务列表（增删改），设置每个任务的基准分，系统自动按 ±20% 波动。

7. **作为家长**，我希望每周看到孩子的成长报告（任务完成率、金币变化等），了解孩子的近况。

8. **作为家长**，我希望可以修改家长自己和孩子的账号密码，设置孩子所在年级（影响数学题难度）。

## 3. 功能规约

### 模块 A：任务打卡系统（核心模块）

| 项目 | 内容 |
|------|------|
| **输入** | 孩子选择当天的任务，点击"打卡" |
| **行为** | 系统按盲盒机制计算金币：`波动 = max(1, floor(基准分 × 20%))`，`实际分 = 基准分 + random(-波动, +波动)`，`金币 = 实际分 × 10` |
| **输出** | 打卡后揭晓金币数，显示庆祝文案 |
| **边界条件** | 同一天同一任务不可重复打卡 |
| **错误处理** | 如果任务已被家长停用，前端隐藏不显示 |

**盲盒机制细节：**
- 孩子在打卡前只看到任务名和"？"符号
- 打卡后揭晓实际金币
- 基准分由家长设置，波动自动 ±20%（向下取整，最低为 1）
- 锁定状态的任务不波动，固定给基准分

**预设任务：** 读书、写字、画画、弹钢琴、跳舞、做数学题。家长可新增、删除任意任务。

### 模块 B：家长管理端

| 项目 | 内容 |
|------|------|
| **输入** | 家长登录后进入管理界面 |
| **行为** | 管理任务、查看报告、管理账号等 |
| **输出** | 配置保存后实时生效 |

**功能列表：**
1. 首次运行引导创建管理员账号
2. 任务管理：增删任务、设置基准分、锁定选项
3. 孩子管理：创建孩子账号、修改密码、设置年级
4. 账号管理：修改自己的密码
5. 成长报告：查看孩子的周统计数据
6. 使用指南：内置折叠式使用说明

### 模块 C：宠物养成系统

| 项目 | 内容 |
|------|------|
| **输入** | 孩子选择互动类型（喂食/玩耍/摸一摸） |
| **行为** | 扣除金币，增加成长值，显示对应的互动文案 |
| **输出** | 宠物反应文案 + 成长进度更新 |

**宠物机制：**
- 可同时养多只宠物，最多 5 只
- 宠物蛋需 200 金币购买，选择动物类型和名字
- 蛋阶段：仅需喂食（8 次）→ 孵化
- 幼年阶段：喂食 + 玩耍（共 23 次喂食 + 10 次玩耍）→ 成年
- 蛋阶段只显示喂食按钮，不显示玩耍/抚摸
- 成年后可放生，放生的宠物记录在"曾经的小伙伴"中
- 成长完全按操作次数，不受真实时间影响

**5 种动物预设：** 小猫、小狗、小兔子、小恐龙、小企鹅（每种有 15 条不同的互动文案）

**宠物图片：**
- 每种宠物幼年 3 张、成年 3 张真实图片
- 用宠物 ID 决定选哪张，同一只宠物永远显示同一张，不同宠物不同
- 蛋阶段使用 🥚 emoji

**消耗：**

| 项目 | 价格 |
|------|:----:|
| 宠物蛋 | 200 金币 |
| 喂食 | 50 金币/次 |
| 玩耍 | 80 金币/次 |
| 摸一摸 | 免费 |

### 模块 D：每日答题系统

| 项目 | 内容 |
|------|------|
| **输入** | 孩子点击"开始答题"，系统出题，输入答案 |
| **行为** | 显示题目 → 输入答案 → 判断对错 → 继续或结束 |
| **输出** | 答对：继续下一题；答错：本轮结束，结算金币 |

**规则：**
- 每天最多 10 轮
- 每轮最多 10 题，答错立即结束本轮
- 金币 = 答对题数 × 10
- 答完 10 题或答错后自动结算

**难度分级（家长在设置中选择孩子的年级）：**

| 年级 | 题目范围 |
|------|---------|
| 幼儿园 | 10 以内加减法 |
| 一年级 | 100 以内加减法，无进位退位 |
| 二年级 | 九九乘除法 + 100 以内加减法 |
| 三年级 | 1000 以内加减法 + 有余数除法 + 两位数乘整十数 |

### 模块 E：勋章墙

| 勋章 | 获得条件 |
|------|---------|
| 🌟 初来乍到 | 完成第 1 个任务 |
| 🌟 坚持不懈 | 连续 7 天完成任务 |
| 🌟 数学小天才 | 数学题累计答对 20 题 |
| 🌟 攒钱小能手 | 累计获得 500 金币 |
| 🌟 宠物达人 | 成功养大 3 只宠物 |
| 🌟 满分一周 | 本周任务完成率 100% |

### 模块 F：成长报告

- 每周自动生成
- 展示内容：任务完成率、金币收支、活跃天数

## 4. 非功能性需求

### 4.1 性能
- 页面加载时间不超过 2 秒
- 同时支持 1-3 个家庭成员使用

### 4.2 安全
- 所有密码使用哈希加密存储
- 密码全系统唯一，防止账号冲突
- 敏感配置通过环境变量加载，绝不硬编码进源码
- 密钥/凭据绝不提交 Git 仓库

### 4.3 可用性
- 界面简洁，孩子端以大按钮和图标为主
- 家长端功能分区清晰
- 首次运行有初始化引导
- 支持 PWA，可添加到手机桌面

### 4.4 可观测性
- 所有关键操作记录日志
- AGENT_LOG.md 记录开发过程中的关键决策

## 7. 凭据与分发设计

### 7.1 凭据安全

- **密码存储**：使用 Werkzeug 的 `generate_password_hash` 进行哈希加密，不存明文
- **密码唯一性**：全系统密码不可重复，防止账号混淆
- **API Key / 敏感配置**：通过环境变量加载
- **.env 文件**：用于本地开发环境，添加至 `.gitignore` 确保不提交
- **Git 安全**：提交前自查，确保仓库中无任何明文凭据

### 7.2 分发方案

**方案一：Docker 容器（满足作业要求）**
- 提供 `Dockerfile`，一条命令构建运行
- CI（GitLab CI）自动构建

**方案二：云端部署**
- 部署到 Railway.app（免费，自动从 GitHub 部署）
- 提供公网访问 URL

**用户使用流程：**
1. 打开浏览器 → 访问链接
2. 首次运行 → 自动跳转到"创建管理员账号"
3. 创建账号后进入家长端 → 创建孩子账号 → 配置任务
4. 孩子登录 → 开始每日打卡

## 8. 技术选型与理由

| 技术 | 选型 | 理由 |
|------|------|------|
| **后端框架** | Python Flask | 轻量、上手快，适合小型 Web 应用 |
| **数据库** | SQLite（WAL 模式） | 零配置，单文件，Python 内置支持 |
| **前端** | HTML + CSS + JavaScript | 无需学习前端框架，纯静态页面 |
| **模板引擎** | Jinja2（Flask 内置） | 服务端渲染，简化前后端交互 |
| **密码加密** | Werkzeug | Flask 生态自带，安全可靠 |
| **容器化** | Docker | 作业要求的分发方式 |
| **云部署** | Railway.app | 免费，GitHub 自动部署 |
| **CI** | GitLab CI | 自动运行测试 |
| **PWA** | Service Worker + Manifest | 支持添加到手机桌面 |

**选型核心思路：** 以你现有的 Python 基础出发，选最简单、最不折腾的技术栈，把精力放在功能逻辑上。

## 10. 风险与未决问题

### 风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 免费云部署无持久化存储 | 每次部署数据库重置 | 本地 Docker 运行数据持久；部署为演示用途 |
